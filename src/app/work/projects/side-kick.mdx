## Overview

**Sidekick** is a mental health platform where users find accountability partners, receive scientifically-backed roadmaps, and get expert help navigating the journey of self-improvement—all within a gamified app. The platform aims to make self-improvement engaging and collaborative by connecting users with others who share similar habits and goals.

I contributed to both the application and backend development, implementing key features and ensuring a seamless user experience.

## Key Features

- **Habit Selection and Pact Creation**: Users choose a habit and duration, after which a pact is created between them and an accountability partner.
- **Daily Tasks and Fingerprint Scanning**: Users receive daily tasks to complete. Both partners must scan their fingerprint to mark the task as completed, promoting mutual accountability.
- **Habit Completion and Progression**: After a predefined period, users finish their current habit and can choose a new one, fostering continuous self-improvement.
- **Admin Dashboard**:
  - **Habit Management**: Admins can create and list habits, which are filtered based on common attributes to match users effectively.
  - **Monitoring Metrics**: Admins can monitor app metrics such as user activity, chosen habits, group texts, and success rates.

## Technologies Used

### App Development

- **Capacitor**: For building cross-platform native apps using web technologies.
- **Ionic**: Utilized for UI components and design.
- **Zustand**: Employed for state management in React applications.
- **Socket.io**: Implemented for real-time messaging between users.
- **Tailwind CSS**: Used for styling the application.

### Backend Development

- **Flask**: Served as the backend framework.
- **Socket.io**: For real-time communication in the backend.
- **Cron Jobs**: Scheduled tasks for user group scheduling.
- **CMS**: For habit listing and creation.

## Challenges and Learnings

- **Real-Time Communication**: Implementing real-time messaging and fingerprint scanning required efficient use of Socket.io to ensure low latency and high reliability.
- **Gamification Elements**: Designing the app to be engaging and motivating involved integrating gamification techniques without compromising the seriousness of mental health topics.
- **State Management**: Managing complex state across the app was handled using Zustand, providing a lightweight and scalable solution.
- **Backend Scheduling**: Used cron jobs for scheduling tasks, such as user group assignments, to automate processes and enhance scalability.

## Outcome

The Sidekick app is still under development, providing users with a supportive community and a structured approach to self-improvement. By combining accountability partnerships with gamified elements, the platform encourages consistent habit formation and personal growth.
